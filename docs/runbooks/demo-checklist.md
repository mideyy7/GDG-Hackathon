# Demo Checklist

## Pre-Demo
1. Telegram bot token validated.
2. GitHub app permissions verified (issues + PRs).
3. Planner approval card renders with Approve/Reject.
4. Generator/Reviewer rewrite loop tested once.
5. Dashboard showing task history from gateway.

## During Demo
1. Start from a fresh issue request in Telegram.
2. Show architecture plan and approval gate before coding.
3. Show GLM chain-of-thought reasoning in Telegram.
4. Show reviewer rewrite event (live or pre-staged backup).
5. Show PR output with changelog + docs.
6. End on the DevClaw dashboard showing the multi-model pipeline.

## Fallbacks
1. Timeout fallback for model calls > 15s.
2. Pre-recorded reviewer rewrite clip.
3. Last-known-good PR link ready to display.
